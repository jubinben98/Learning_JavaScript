Call Stack

- Call stack maintains the order of execution of execution context.

- For the current example:

 

GEC

 

- After the [Line 6] square(2) is detected, a new execution context is added to the stack.

 

EC1

GEC

 

- After the EC1 is completed it is pop-out from stack.

 

GEC

 

- After the [Line 7] square(4) is detected, again a new execution context is added to the stack.

 

EC2

GEC

 

- After EC2 is executed EC2 is pop-out and since GEC is executed, the main GEC will also be executed and our stack will be empty.
